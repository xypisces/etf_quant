## Why

我们需要为现有的 ETF 量化交易系统（特别是回测和策略成果）提供一个直观的、易于操作的可视化界面。目前系统具备了底层的回测和策略逻辑，但缺乏友好的前端展示。通过使用 Streamlit 快速搭建轻量级 Web 界面，不仅可以大幅降低使用门槛，还能方便地进行策略参数配置、回测结果可视化、行情监控以及轮动池管理。这使得量化分析过程更加高效直观，同时也为系统的模块化扩展奠定了基础（未来可灵活替换底层图形库）。

## What Changes

- 引入 **Streamlit** 作为前端框架，搭建纯 Python 的轻量级 Web 应用程序。
- 新增 **策略回测面板**，支持品种选择、策略选择、参数配置以及一键运行回测，并展示净值曲线、买卖点和绩效指标。
- 新增 **ETF 行情看板**，提供多品种涨跌幅排行和走势叠加对比功能。
- 新增 **轮动池管理模块**，用于管理自选 ETF 池并查看相关的动量排名等信息。
- 新增 **回测历史记录功能**，保存并展示历史回测结果，支持基本的可视化对比分析。
- 设计上采用**模块化接入**，确保可视化层与底层逻辑解耦，未来需要时可以轻松替换为其他图形渲染方案。

## Capabilities

### New Capabilities

- `dashboard-ui`: 基于 Streamlit 的量化交易系统主界面框架，负责整体布局、导航和模块调度。
- `backtest-panel`: 交互式的策略回测配置与结果展示组件，集成运行回测、展示曲线、买卖点标记和绩效统计。
- `market-monitor`: ETF 行情监控组件，用于展示涨跌幅排行和多品种走势叠加。
- `pool-manager`: 轮动池与自选 ETF 池管理组件，包含动量排名计算的展示。
- `history-viewer`: 历史回测结果的持久化读取与对比展示界面。

## Impact

- **新增依赖**：项目中将引入 `streamlit` 库以及可能用于绘图的配合库（如 `plotly` 或 `altair` 等，具体在设计阶段确定）。
- **新增模块**：将在系统中建立独立的前端/可视化目录（如 `app/` 或 `frontend/`），存放所有 UI 相关的代码。
- **解耦设计**：现有的 `engine`、`metrics` 等后端模块将被作为 API 或服务层被 UI 层调用，确保后端逻辑不受前端变动的影响。
- **系统入口**：可能需要新增一个启动脚本（如 `run_dashboard.py`）来专职启动 Streamlit 服务。
